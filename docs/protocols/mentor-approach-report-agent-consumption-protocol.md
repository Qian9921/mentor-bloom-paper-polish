# Mentor Approach Report Protocol v0.1

目标：在真正进入导师式改稿之前，先给出一份“当前稿件与导师标准的趋近度报告和评分”。

---

## 1. 为什么这一步重要

很多系统直接开始改稿，但没有先回答：

- 现在离导师标准还有多远？
- 最重要的差距在哪里？
- 这轮改稿最该追求什么？

`mentor_approach_report` 的作用，就是在改稿前先给 agent 一个清晰目标。

---

## 2. 分层原则

- script：根据 xray 和 mentor profile 生成结构化 approach report
- agent：消费这个报告，把它当作 revision target

---

## 3. 报告应回答的问题

1. 当前稿件整体与导师标准的趋近度是多少？
2. 最强的部分是什么？
3. 最大的写作差距是什么？
4. 这轮改稿最该优先追哪个目标？

---

## 4. 使用原则

`mentor-polish` 在 canonical flow 中，应该优先读取这份报告，再开始 pass-based rewriting。
